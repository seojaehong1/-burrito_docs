프로젝트 주제 : 커피전문점 마이크로서비스 통합 플랫폼 (TORY COFFEE)

개요
본 프로젝트는 마이크로서비스 아키텍처(MSA)를 기반으로 각 비즈니스 기능을 독립적인 서비스로 분리하여 개발,배포 가능한 
커피전문점 통합 관리 플랫폼을 구죽하는 것을 목표로 합니다. 
Spring Cloud(Eureka, Gateway)를 활용해 서비스 디스커버리 및 단일 진입점을 구성하여 관리 효율성을 높였으며, 특히 RabbitMQ를 도입한 이벤트 기반(Event-Driven) 비동기 메시지 처리를 통해 주문과 재고 서비스 간의 결합도를 낮추고 시스템 안정성을 확보했습니다. 모든 서비스는 Docker 컨테이너 환경에서 AWS EKS(Kubernetes)로 배포되어, 트래픽에 따라 스스로 확장되는 클라우드 네이티브 시스템을 지향합니다.

설계의 주안점
- MSA 기반의 서비스 독립성 확보: 기능별 서비스 분리 및 개별 DB 설계를 통한 독립적 배포 및 유지보수 환경 구축
- 서비스 거버넌스 및 라우팅 최적화: Spring Cloud Eureka와 Gateway를 활용한 동적 서비스 등록 및 지능형 부하 분산(Load Balancing)
- 이벤트 기반 비동기 데이터 정합성 유지: RabbitMQ를 활용하여 주문 발생 시 재고가 실시간 반영되는 비동기 통신 구조 구현
- 클라우드 인프라 가용성 극대화: AWS EKS 내 HPA(Horizontal Pod Autoscaler) 설정을 통한 트래픽 대응 자동 확장 시스템 구축
- 보안 및 인증 통합: API Gateway 계층에서 JWT 인증을 처리하여 마이크로서비스 간 보안 통신 및 사용자 권한 관리 통합

응용 분야
- 대규모 프랜차이즈 통합 주문·재고 관리: 지점별 트래픽 변화가 심한 대형 커피 전문점의 안정적인 주문 처리 및 실시간 재고 추적 시스템
- 이벤트 기반 F&B 운영 플랫폼: 주문 - 재고 차감 - 상태 관리 등 각 단계가 비동기 메시지로 연결되는 외식 업종의 운영 모델
- 고가용성 클라우드 네이티브 서비스 모델: 트래픽에 따라 자원을 자동 조절해야 하는 이커머스 및 예약 시스템의 아키텍처 표준
- MSA 및 DevOps 교육/테스트 베드: 서비스 분리, 컨테이너 오케스트레이션(K8s) 실습을 위한 기술 검증 플랫폼

사용 기술
- 개발환경 : Windows 10, IntelliJ IDEA
- 개발언어 : Java (JDK 17), JavaScript
- 개발도구 : Git / GitHub, Gradle
- 프레임워크 : Spring Boot 3.1.5, Spring Cloud (Eureka, Gateway), Thymeleaf 
- 보안 : Spring Security, JWT 인증
- 데이터베이스 : H2(로컬), MySQL 8.0 (Kubernetes StatefulSet)
- 메시지 큐 : RabbitMQ 3.x (Spring AMQP)
- 컨테이너 : Docker
- 오케스트레이션 : Kubernetes (AWS EKS)
- 로드밸런서 : AWS ALB

조원
서재홍, 박윤호, 이주희, 하성호
